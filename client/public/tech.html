라이브 데모는 게임 서버 없이 GIWA 체인만으로 동작합니다 — 노점·길드·던전 기록은 컨트랙트에, 아바타 위치는 프레즌스 비컨(이벤트)으로. "가치는 온체인, 존재감은 비컨." 자동 시연은 데모 URL에 ?showcase=1, 또는 클론 후 showcase.cmd

기와장터는 GIWA 체인(이더리움 L2, OP Stack) 위의 3D 소셜 플랫폼입니다. 지갑이 아바타가 되고, P2P 거래가 노점이 되고, 신원 인증이 상인의 뱃지가 됩니다. 설계 원칙은 하나입니다 — 서버는 공간만 담당하고, 가치는 전부 체인 위에서 움직인다.

1시스템 아키텍처

클라이언트 (브라우저) React Three Fiber 3D 마을
viem 지갑·컨트랙트 호출
Colyseus.js 실시간 동기화
⇄WebSocket 15Hz
Colyseus 룸 서버 아바타 위치·이모트 중계
노점 레지스트리(영속)
이름 중복 해소·유휴 정리
⇄JSON-RPC
GIWA Sepolia (L2) GiwaMarketV2 (상거래)
DojangScroll (신원 인증)
UPNameRegistry (이름)

※ 결제·소유·인증은 클라이언트가 체인과 직접 통신합니다. 서버는 자금·키를 일절 다루지 않는 비수탁(non-custodial) 구조이며, 서버가 꺼져도 자산은 안전합니다. 정적 데모 모드에서는 서버 없이 NPC를 로컬 시뮬레이션하고 온체인 기능은 동일하게 동작합니다.

3핵심 플로우: 노점 구매

  1. 손님이 노점 클릭 → 상품 선택 → GiwaMarketV2.buy(판매자, 아이템ID) 호출 (viem, 클라이언트 직접 서명)
  2. 컨트랙트가 리스팅 가격과 정확히 일치하는 금액만 수락 → 대금은 에스크로에 보관
  3. 같은 트랜잭션에서 ERC-1155 쿠폰 토큰이 구매자 지갑으로 민팅 (tokenId = keccak(판매자, 아이템ID))
  4. Purchased 이벤트가 온체인 영수증 — 클라이언트는 tx 해시만 서버에 전달, 서버는 형식 검증 후 판매 피드 브로드캐스트
  5. 구매자가 쿠폰함에서 정산 확정 → confirm() → 판매자에게 대금 전달 (미확정 시 24시간 후 release() 가능)
GIWA의 1초 블록 덕분에 1~5단계가 대화 템포(1~2초) 안에 끝납니다 — "게임 속 상점"의 체감을 실제 온체인으로 구현한 핵심 근거입니다.

4공간 설계: 노점의 영속성

노점은 주인이 접속을 끊어도 마을에 남습니다(레지스트리·가격 모두 온체인). 소셜 플랫폼의 최대 리스크인 콜드 스타트를 유저 생성 콘텐츠(노점)가 공간을 채우는 구조로 풉니다 — 동시접속이 적어도 저잣거리는 상점으로 북적입니다. 고전 MMO 저잣거리 문화의 구조적 재해석입니다.

5기술 스택 & 품질

스택 클라이언트: React Three Fiber(three.js) · zustand · viem · Colyseus.js
서버: Node.js · Colyseus (15Hz 스냅샷, 하트비트 유휴 정리, 자동 재접속)
컨트랙트: Solidity 0.8.36 (ERC-1155 자체 구현 포함, 외부 의존성 0)
배포: GitHub Actions → GitHub Pages (정적 데모 모드)
검증 E2E 스모크 테스트 6종 — 실시간 동기화 / 선물 실전송 / 노점 서버 플로우 / 컨트랙트 실거래(에스크로 보관·쿠폰 민팅·가격 강제 revert·이중 정산 거부) / Dojang 조회 / UP.ID 조회. 모든 주장에 온체인 트랜잭션 증거가 있습니다.
신뢰 설계 비수탁: 서버는 자금·개인키 미접촉 · 가격은 체인이 강제 · 영수증은 이벤트 로그 · 서버 수신 메시지는 tx 해시 형식·발신자 검증 후에만 중계 (표시용 데이터는 체인이 진실 소스)
규제 안전선 게임 플레이 보상은 양도 불가(소울바운드 설계) — 사행성(환전) 고리 차단 · 거래 대상은 코스메틱·쿠폰 등 재화만 (금융상품·오더북 없음) · 포지셔닝은 게임이 아닌 아바타 소셜 플랫폼

6GIWA Tech Use — 사용 중인 체인 기능

① Dojang Verified Address — isVerified(주소, 발행자 ID) 온체인 조회로 상인 인증 뱃지 표시. ② UP.ID(Upbit Web3 Names) — 주소→이름 역방향 조회로 아바타·노점 이름표. ③ 1초 블록 — 구매·비컨 확정 지연의 상한, 던전 주간 시드(블록 해시 핀)의 시간 해상도. ④ OP Stack L2 가스 단가 — 프레즌스 비컨 (이벤트 전용 tx, 회당 ~2µETH)을 실용적으로 만드는 전제. ⑤ 웹 기반이라 월렛 웹뷰 탑재가 가능한 구조.

7TODO — 기술 확장 로드맵

현재 아키텍처는 "가치는 온체인, 존재감은 비컨" — 게임 서버 0으로 동작한다. 다음 단계는 서버를 신뢰가 아닌 성능을 위해 다시 들이는 것이다.

  • 읽기 캐시/인덱서 — 접속자가 늘면 모든 클라이언트가 RPC를 직접 폴링하는 구조는 레이트리밋에 막힌다. 계획: 경량 인덱서(Cloudflare Worker 또는 기존 Colyseus 서버 재활용)가 노점·길드·비컨 이벤트를 구독·캐시하고, 클라이언트는 캐시를 우선 읽되 캐시가 죽으면 자동으로 RPC 직접 폴링으로 폴백 — 서버는 있으면 빠르고, 없어도 마을은 돈다.
  • 실시간 승격(하이브리드 프레즌스) — 서버가 살아 있으면 이동·이모트를 Colyseus 릴레이(15Hz)로 승격해 부드러운 실시간, 서버가 없으면 지금의 2초 비컨+데드레커닝으로 자동 강등. 위치는 어차피 가치가 아니므로 어느 쪽이든 체인 기록과 무관하다.
  • 비컨 압축·배치 — 다수 동시 접속 시 이동 델타 인코딩과 배치 전송으로 비컨 tx 수를 줄인다 (또는 세션키 기반 집계 릴레이).
  • 던전 확장 — 문 결과 종류 추가(상인방: 쿠폰 할인, 보물방: 칭호 조각), 시즌제 아카이브(주간 기록의 온체인 보존), 길드 대항 이벤트.
  • 실브랜드 기프티콘 — 쿠폰 소각(redeem) 이벤트를 기프티콘 애그리게이터 API 발급과 연결 — 온체인 사용 증빙이 곧 바코드 스캔.
  • 분쟁 v2 — 현재는 구매자 신고(7일 연장) + 판매자 환불. 다음: 기한 내 미해결 건의 커뮤니티 중재(Dojang 인증 지갑 배심) 실험.
