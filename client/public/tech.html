라이브 데모는 게임 서버 없이 GIWA 체인만으로 동작합니다 — 노점·길드·던전 기록은 컨트랙트에, 아바타 위치는 프레즌스 비컨(이벤트)으로. "가치는 온체인, 존재감은 비컨." 자동 시연은 데모 URL에 ?showcase=1, 또는 클론 후 showcase.cmd

기와장터는 GIWA 체인(이더리움 L2, OP Stack) 위의 3D 소셜 플랫폼입니다. 지갑이 아바타가 되고, P2P 거래가 노점이 되고, 신원 인증이 상인의 뱃지가 됩니다. 설계 원칙은 하나입니다 — 서버는 공간만 담당하고, 가치는 전부 체인 위에서 움직인다.

1시스템 아키텍처 — 서버리스 풀온체인

게임 서버가 없습니다. 브라우저가 GIWA 체인과 직접 읽고 씁니다. 거래·소유·기록은 컨트랙트가 들고, 실시간 위치만 저장 없는 이벤트(비컨)로 흘립니다. 그래서 두 방문자는 서버 없이 서로를 보고 거래합니다.

※ 자금·키는 어떤 서버도 다루지 않는 비수탁 구조. 선택적 Colyseus 서버는 성능(위치 릴레이·읽기 캐시) 용도로만 붙일 수 있고, 없어도 마을은 돕니다 (스케일 로드맵 참고).

3핵심 플로우 — 그림으로

(a) 노점 구매 + 에스크로 — 가격은 체인이 강제하고, 대금은 확정까지 컨트랙트가 보관합니다.

(b) 던전 — 로컬 즉시 플레이 + settleRun 재계산 검증

문을 열 때마다 tx를 보내지 않고 귀환 한 번으로 전 과정을 검증합니다 — 서버판보다 빠르고, 서버 없이도 트러스트리스. (결정론 결과의 한계는 AI Self-Audit에 공개)

(c) 프레즌스 — 2초 비컨 + 데드레커닝

위치는 분쟁 가치가 없어 체인 상태로 두지 않습니다 — 저장 없는 이벤트라 비컨 1회 ≈ 2µETH. 악의적 좌표는 수신 시 월드 반경으로 클램프, 잔액 부족 시 자동 관전 모드(읽기는 무료).

4재미 설계 — 왜 다시 오는가

지갑이 기록이라면 마을은 장소여야 합니다. 장소가 이기려면 세 가지가 필요합니다: 머무를 이유·만날 시간·자랑할 것. 네 가지 시스템이 이 축을 온체인으로 채웁니다.

🔥 모닥불 (머무름) 광장 모닥불에 함께(2명 이상) 앉으면 10분 창마다 온기가 쌓입니다. 혼자서는 안 됩니다 — "함께 있음"의 온체인 증명 (GiwaHearth).
🏮 장날 (시간 약속) 매주 토 21시(KST) 장날엔 온기 2배 — 컨트랙트가 요일을 판정합니다. 모두가 같은 시간에 모일 이유가 밀도를 만듭니다.
🚩 길드 깃발 (소유) 주간 등반 상위 길드의 깃발이 광장에 게양됩니다. 개인 배지를 넘어 공간에 보이는 순위 — 싸우고 자랑할 이유.
🎨 문양 공방 (경제) 유저가 8×8 문양을 그려 온체인 등록·판매·착용. 창작물 거래는 보상이 아니므로 안전선과 양립 — 경제의 상류가 생깁니다.

노점은 주인이 접속을 끊어도 마을에 남습니다(레지스트리·가격 모두 온체인) — 콜드 스타트를 유저 생성 콘텐츠가 공간을 채워 풉니다. 참고 결: 마비노기 모닥불(함께 있는 것 자체가 콘텐츠), 리니지 혈맹(소유의 정치), 로블록스/마크(플랫폼은 도구·유저는 콘텐츠).

6GIWA Tech Use — 사용 중인 체인 기능

① Dojang Verified Address — isVerified(주소, 발행자 ID) 온체인 조회로 상인 인증 뱃지 표시. ② UP.ID(Upbit Web3 Names) — 주소→이름 역방향 조회로 아바타·노점 이름표. ③ 1초 블록 — 구매·비컨 확정 지연의 상한, 던전 주간 시드(블록 해시 핀)의 시간 해상도. ④ OP Stack L2 가스 단가 — 프레즌스 비컨 (이벤트 전용 tx, 회당 ~2µETH)을 실용적으로 만드는 전제. ⑤ 웹 기반이라 월렛 웹뷰 탑재가 가능한 구조.

7나중 작업 (P2) — 스케일 로드맵

현재 아키텍처는 "가치는 온체인, 존재감은 비컨" — 게임 서버 0으로 동작하고, 수십 명 규모의 밀도 있는 광장에 최적화돼 있다. 병목은 체인이 아니라 ① 공개 RPC 폴링 (유저 수 × 2.5초) ② 클라 렌더 O(N) ③ 전체 순회 뷰다. 다음 단계는 서버를 신뢰가 아닌 성능을 위해 되들이는 것이다.

구역 샤딩 (세계관과 맞물리는 확장) — 저잣거리를 여러 골목(비단골목·먹자골목…)으로 나누면 각 광장은 수십 명 밀도를 유지하면서 전체는 확장된다. 밀도 낮은 대광장보다 좁고 북적이는 골목이 사회적으로도 낫다. 아래 인덱서·하이브리드 릴레이와 함께 가면 수백~수천 동접 경로가 보인다. (메인넷 가스에서의 비컨 빈도 적응은 열린 문제)
  • 읽기 캐시/인덱서 — 접속자가 늘면 모든 클라이언트가 RPC를 직접 폴링하는 구조는 레이트리밋에 막힌다. 계획: 경량 인덱서(Cloudflare Worker 또는 Colyseus 서버)가 노점·길드·비컨 이벤트를 구독·캐시하고, 클라이언트는 캐시를 우선 읽되 캐시가 죽으면 자동으로 RPC 직접 폴링으로 폴백 — 서버는 있으면 빠르고, 없어도 마을은 돈다.
  • 실시간 승격(하이브리드 프레즌스) — 서버가 살아 있으면 이동·이모트를 Colyseus 릴레이(15Hz)로 승격해 부드러운 실시간, 서버가 없으면 지금의 2초 비컨+데드레커닝으로 자동 강등. 위치는 어차피 가치가 아니므로 어느 쪽이든 체인 기록과 무관하다.
  • 비컨 압축·배치 — 다수 동시 접속 시 이동 델타 인코딩과 배치 전송으로 비컨 tx 수를 줄인다 (또는 세션키 기반 집계 릴레이).
  • 던전 확장 — 문 결과 종류 추가(상인방: 쿠폰 할인, 보물방: 칭호 조각), 시즌제 아카이브(주간 기록의 온체인 보존), 길드 대항 이벤트.
  • 실브랜드 기프티콘 — 쿠폰 소각(redeem) 이벤트를 기프티콘 애그리게이터 API 발급과 연결 — 온체인 사용 증빙이 곧 바코드 스캔.
  • 분쟁 v2 — 현재는 구매자 신고(7일 연장) + 판매자 환불. 다음: 기한 내 미해결 건의 커뮤니티 중재(Dojang 인증 지갑 배심) 실험.
