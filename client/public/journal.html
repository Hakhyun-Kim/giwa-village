두 번의 밤, 한 번의 아침

실제 커밋 시각을 3시간 단위 8개 구간으로 나눠 보면, 몰입이 언제 몰렸는지가 그대로 드러난다. 막대 위에 마우스를 올리면 구간별 세부 수치가 뜬다.

낮·저녁 시간대 자정 전후 (23시~새벽 1시)

7/18 밤(21–24시)과 7/19 밤(21–23시) 두 번의 야간 몰입 구간이 뚜렷하다. 그 사이 새벽 1시부터 7시까지는 커밋이 없다 — 유일하게 완전히 빈 6시간, 짧은 휴식이다. 가장 큰 봉우리는 7/19 낮(12–15시) — 랜덤박스·모닥불·문양 공방 세 기능이 한 번에 쏟아진 구간. 7/20·7/21은 리듬이 확연히 다르다. 하루 18개까지 갔던 커밋이 6개·4개로 줄었는데, 같은 기간 삽입 라인은 5,100줄 넘게 늘었다 — 커밋 하나가 무거워졌다는 뜻이다. 기능을 쌓는 단계에서 기반을 다지는 단계로 넘어간 흔적.

코드베이스

코드가 자라난 모양

공개 저장소 32개 커밋의 누적 삽입 라인. 점을 지나가면 그 순간 무슨 커밋이었는지 뜬다. 초반의 급경사는 최초 스캐폴드(9,845줄)와 풀온체인 전환(+2,341줄) — 이후로는 완만하지만 꾸준한 계단이다.

현재 코드베이스: 컨트랙트 1,601줄(Solidity 12개 파일, 현행 10종) · 클라이언트 11,060줄(56개 파일) · SDK 200줄 — 총 122개 파일 추적 중.

핵심 · 프롬프트의 진화

무엇을 배웠는가

이 문서의 진짜 요지는 커밋이 아니라 질문이 달라진 방식이다. 아래는 실제로 오간 요청을 시간순 그대로 발췌한 것 — 각 단계가 드러내는 감각을 짧게 덧붙였다.

0107/18 저녁
신뢰 기반 위임

"다음 할 거 진행해줘"

아직은 "무엇을 만들지"가 아니라 "무엇부터 할지"를 묻는 단계. 저장된 우선순위를 믿고 큰 그림을 맡기며 시작했다.

0207/18 밤
관찰 가능한 증거

"실시간으로 키조작 없이 서버/클라이언트 열고, 상점 열고 등등 작업 → 테스터가 별다른 설명 없이 보고만 있어도 필요한 작업을 바로 하게"

말로 된 결과 보고가 아니라, 무조작으로 굴러가는 걸 직접 보고 싶다는 요구. 이후 모든 기능에 자동 시연(showcase)·E2E 검증이 따라붙는 습관의 출발점.

0307/19 새벽
제품 경계·리스크 감각

"audit 을 스스로 한다고 생각하고 코드 보고 한번 수정필요하면 해줘. tech 문서등에서는 라그나로크 언급할 필요없을듯 … 업비트와 협업이 필요하거나 서버가 필요한 부분은 배제하려고 해. 외부 의존도를 줄여서 최대한 개발"

기능을 더 얹는 게 아니라 무엇을 하지 않을지를 먼저 그었다. 자체 감사, 상표/IP 리스크, 외부 의존성 — 서로 다른 세 종류의 리스크를 한 호흡에 짚어낸 지점.

0407/19 낮
시스템을 직접 설계

"위치는 1초에 1번 정도 받아서 방향 예측해서 움직여 보면 어떨까? 대충 어느 상점으로 향한다는 걸로 받아도 좋고."

이건 요청이 아니라 설계다 — 멀티플레이어 게임의 표준 기법인 데드레커닝(위치 예측 보간)을 스스로 제시했다. 온체인 제약(가스비·트랜잭션 빈도) 위에서 20년 게임엔진 경력이 정확히 재조준된 순간.

0507/19 낮
비교 설계 · 제품 비평

"왜 지갑에서 친구 목록 대신 여기에 와서 즐기고 놀고 하는지 … 마비노기 모닥불 같은 예시도 검토해주고 … 리니지 온라인 … 로블록스나, 마인크래프트 추가로 더 필요한 내용들도 검토해줘"

기능이 아니라 "왜 다시 오는가"를 물었다. 세대가 다른 MMO 넷을 온체인이라는 새 제약 위에 겹쳐 놓고 재미의 근원을 해부 — 결과로 모닥불·장날·길드 깃발·도깨비 토벌이 나왔다.

0607/19 저녁
인프라 재정의

"길드시스템을 좀 더 확장해서 온체인 저장되어있으니까 외부에서도 길드원들의 정보를 읽을 수 있는등 확장 가능한 시스템으로 … 시스템적인 구현 SDK 구현에 가까울수도"

온체인 저장이 곧 "누구나 읽을 수 있다"는 함의를 스스로 도출하고, 정확한 이름(SDK)까지 붙였다. GiwaProfile 애그리게이터와 @giwa-village/sdk로 이어졌다.

0707/19 밤
정보 설계 · UX 감각

"UI 등이 편리한지, 줄일부분은 없는지 … RPG 의 퀘스트 형식 어떨까?"

기능이 쌓이며 생긴 진입장벽(HUD 7버튼)을 스스로 감지하고, 익숙한 UX 어휘로 구체적 해법을 제시했다. HUD는 4버튼으로, 온보딩은 촌장의 부탁 7단계 퀘스트로 정리됐다.

0807/20
회고 · 지도 갱신 습관

"문서 정리된 건지? 아니면 정리해줘~" — 그리고 "내가 어떻게 공부하고, 학습하고, 결과적으로 좋아졌는지 중점"

코드보다 지도(문서)가 먼저 낡지 않았는지 확인하는 습관, 그리고 그 과정 자체를 되짚어보려는 습관. 이 문서 자체가 그 여덟 번째 단계다.

0907/21 오전
외부 참조 · 취사선택

"최근에 이런 걸 만들었는데, 참고해서 giwa 에 넣을만한 게 있을까? 분석하고 리스트로 정리해줘."

남이 만든 것을 가져와 그대로 따라 하지 않고 무엇이 이 마을에 맞는지 골라내는 단계. 여기서 나온 가장 큰 수확은 기능이 아니라 관점이었다 — 다른 곳에서 "에이전트와 사람을 동등하게 대한다"가 설계 목표라면, 지갑과 서명만으로 들어오는 이 마을에서는 그게 이미 구조적 성질이라는 것. 특권 API를 안 만든 게 아니라 만들 자리가 없다. 그 관점 위에 MCP 서버와 자기 지갑을 든 LLM 상인 NPC를 얹었고, 값은 모델이 제안하되 하한선은 코드가 강제하도록 안전선을 코드 쪽에 두었다.

10지금
제약을 설계로 바꾸기

"테스트 가스비를 쓰니까 테스트가 어려운 면이 있는데 … 가스비 없는 로컬 서버 테스트 / 가스비 있는 테스트 이렇게 나누어주고 … local giwa backend 설정등이 가능한가?"

포셋이 24시간에 0.005 ETH라는 제약을 불평이 아니라 테스트 설계 문제로 바꿔 물었다. 답은 로컬 EVM 노드를 GIWA와 같은 chain-id(91342)로 띄우는 것 — 코드의 체인 가드가 그대로 통과하니 프로덕션과 같은 경로로 도는 진짜 E2E인데 가스만 무한하다. (GIWA 자체 노드는 500GB로 실제 테스트넷을 동기화하는 구조라 개발용으로는 맞지 않았다.) 결과적으로 검증 25건이 체인 없이 1초 안에 돌고, 컨트랙트 E2E도 무료가 됐다. "어떻게 만들까"에서 "어떻게 계속 확인할까"로 질문이 옮겨간 지점이다.

맺음

70시간 동안 바뀐 것은 코드량이 아니라 질문의 위치다. 초반엔 "다음에 뭘 할까"를 맡겼고, 중반엔 "이건 하지 말자"는 경계를 그었고, 후반엔 "이게 왜 재밌어야 하는가"를 스스로 따졌다. 라그나로크에서 시작해 마비노기·리니지·로블록스를 가로지른 검토, 그리고 온체인 저장이 SDK가 된다는 통찰은 20년 게임엔진 경력이 새로운 제약(가스비, 소울바운드, 서버 없는 신뢰) 위에서 다시 작동한 결과였다.

마지막 하루가 특히 그랬다. 남이 만든 것을 보고 베낄 것과 골라낼 것을 구분했고, 포셋 제약을 불평 대신 테스트 설계 문제로 바꿔 물었다. 기능을 쌓는 속도는 느려졌지만 커밋 하나의 무게는 늘었다 — 만드는 단계에서 지탱하는 단계로 넘어가는 구간이 git 기록에 그대로 남았다.
